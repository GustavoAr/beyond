    Install

    To install beyond.js will simply go to a terminal and use the command npm for this.
    copy 
    # Once released in the npm
$ npm install beyond --save
    An alternative is to specify the dependency in the package.json file.
    copy 
    {
  "name": "hello-world",
  "version": "0.0.1",
  "description": "BeyondJS says ...",
  "main": "server.js",
  "license": "MIT",
  "repository": "github:beyondjs/hello-world",
  "dependencies": {
    "beyond": "github:socites/beyond"
  },
  "engines": {
    "node": ">= 0.12.7"
  },
  "scripts": {
    "start": "node server.js"
  }
}

    The basic structure of this project is as shown below. First the file package.json, and server.js file, which we detail below. Then we have the code directory, where will all the application code. For this particular example, it is just the index.html file and a module named hello with their respective files.

    Setup the Server

    In server.js file is where the importation of BeyondJS is made and a variable called beyond which type is Beyond is created passing an object, where the following elements are defined. As can be see, in the object that it's passed to Beyond is defined application called, also, application, which is in its version 0.1 and define an static file called index.html. Then, run the server with it's start method.

    copy 
    (function (Beyond) {
    "use strict";

    let beyond = new Beyond({
        'applications': {
            'application': {
                "version": "0.1",
                "static": ["index.html"]
            }
        }
    });

    beyond.start();

})(require('beyond'));

    Module Configuration.

    In the module.json file, it's defined two things, the files that the module uses and the route to which responds. Note how files are organized by type, either html, js or css.

    copy 
    {
  "code": {
    "html": {
      "files": [
        "click.html"
      ]
    },
    "js": {
      "files": [
        "hello.js"
      ]
    }
  },
  "route": "/hello-world"

    In the files array of each file type, is the route relative, like in this case the files are in the same directory that module.json, just put the name of the file itself.

    Run the Applicaction

    At this point, we have a small functional web application. For run the server go to a terminal and execute the command node server.js. Some consideration about the URL.

    http://localhost:{port}/{name of application}/{language}/{route}

    Where {port} is defined in the object passed to Beyond.JS, if it not, by default is 3010. The {name of application} as well is defined in the object, in this case the {name of application} is application. .Since BeyondJS is a framework oriented to multi-language applications, by default it expect that {language} been passed through the URL. If not defined by default is english code, en. And the last, {route}, that was defined in the module.json file, in this case the route is /hello-world. Recap, the URL for this example must be.

    http://localhost:3010/application/en/hello-world
